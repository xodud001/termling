# SPEC.md — Termling

Claude Code 토큰 사용량으로 성장하는 터미널 수집형 펫 게임의 단일 소스 스펙.
코드와 이 문서가 충돌하면 이 문서를 따른다. 규칙을 바꾸려면 코드보다 이 문서를 먼저 수정한다.

모든 수치는 기본값이며 `curve.json`으로 오버라이드할 수 있다. 본문에서 `curve.json` 키를 함께 표기한다.

---

## 1. 세계관

터미널 속에는 "털링(Termling)"이라 불리는 디지털 정령들이 산다. 이들의 양식은 토큰 —
개발자가 Claude Code로 일하며 소비하는 토큰이 곧 털링의 에너지다. 개발자가 일할수록 털링은 자란다.

- 모델 비종속: 어떤 모델을 쓰든 토큰량만 본다. 모델명/티어를 읽는 로직은 금지.
- 톤: 가볍고 귀엽게. 개발자 밈(비트, 커서, 셸, 버그, 데몬, 커널)을 크리처 모티프로 사용.

## 2. 크리처

1차 스코프는 총 6종. 종 추가는 스코프 밖(9장).

| ID | 표시명 | 컨셉 | 획득 경로 | 알 가중치 |
|---|---|---|---|---|
| `bitty` | 비티 | 비트 조각 정령 | 스타터 | — |
| `cursy` | 커시 | 깜빡이는 커서 정령 | 스타터 | — |
| `shelly` | 셸리 | 셸 껍질 정령 | 스타터 | — |
| `buggsy` | 버그시 | 벌레 정령 | 알 | 60 (common) |
| `daemo` | 데모 | 백그라운드 데몬 정령 | 알 | 30 (uncommon) |
| `kern` | 컨 | 커널 심층 정령 | 알 | 10 (rare) |

- `init` 시 스타터 3종 중 1종 선택. 선택하지 않은 스타터 2종은 1차 스코프에서 획득 불가.
- 야생 3종은 알 부화로만 획득(5장).
- 알 가중치 키: `egg.speciesWeights`

## 3. 경험치와 레벨

### 3.1 경험치 환산

- **1,000 토큰 = 1 XP.** (`xp.tokensPerXp: 1000`)
- 토큰은 statusline이 stdin JSON에서 읽은 **입력+출력 합산**. 캐시 read/write를 구분하지 않고 제공되는 토큰 필드를 모두 합산한다.
- XP 적립 대상은 **활성 크리처 1마리뿐**이다. 비활성 크리처는 성장하지 않는다 (알은 예외, 5장).
- XP는 정수. 환산 후 소수점 이하는 토큰 잔여량으로 이월한다 (버리지 않는다).

### 3.2 레벨 곡선

- 레벨 n → n+1에 필요한 XP: `ceil(base × n^exponent)`
  - `xp.base: 10`, `xp.exponent: 1.5`
  - 예: Lv.1→2 = 10 XP, Lv.10→11 = 317 XP, Lv.50→51 = 3,536 XP, Lv.99→100 = 9,851 XP
- **만렙 100.** (`xp.maxLevel: 100`) 만렙 후 XP는 누적 기록만 하고 레벨은 오르지 않는다.
- Lv.100 도달 누적 약 40만 XP ≈ 4억 토큰. 헤비 유저 기준 2~3개월을 목표 체감으로 잡는다.

## 4. 진화

진화는 레벨업 시점에 자동 판정된다. 단계는 ASCII 아트 5단계와 1:1 대응.

| 단계 | 레벨 구간 | 명칭 |
|---|---|---|
| 1 | 1 ~ 9 | 유생 |
| 2 | 10 ~ 24 | 성장기 |
| 3 | 25 ~ 39 | 성숙기 |
| 4 | 40 ~ 69 | 완전체 |
| 5 | 70 ~ 100 | 초월체 |

- 진화 관문 레벨: **10 / 25 / 40 / 70** (`evolution.gates: [10, 25, 40, 70]`)
- 관문을 건너뛰는 레벨업(한 번에 여러 레벨)이 발생해도 최종 레벨 기준 단계로 즉시 반영하며, 통과한 관문마다 `evolve` 이벤트를 각각 기록한다.

## 5. 알과 조우

### 5.1 조우 판정

- 활성 크리처가 XP를 얻는 동안 **10,000토큰(10 XP) 누적마다 1회** 조우 굴림. (`egg.rollPerTokens: 10000`)
- 굴림당 알 획득 확률 **2%**. (`egg.encounterRate: 0.02`) 기대값상 약 50만 토큰마다 알 1개.
- **미부화 알은 동시에 1개만** 보유할 수 있다. 알 보유 중에는 조우 판정을 하지 않는다.

### 5.2 부화

- 알은 활성 크리처와 **동시에** 같은 XP를 받는다 (활성 크리처의 몫을 나누지 않는다).
- 누적 **50 XP(5만 토큰)** 도달 시 즉시 부화. (`egg.hatchXp: 50`) — 가벼운 세션 1~2회 분량.
- 부화 종은 획득 시점이 아닌 **부화 시점에** 가중 추첨한다(2장 가중치).
- 부화한 크리처는 Lv.1, 활성 크리처는 바뀌지 않는다 (`switch`로 교체).

### 5.3 중복 부화

- 이미 보유한 종이 부화하면: 도감에 부화 횟수 +1 기록, **활성 크리처에게 보너스 500 XP**. (`egg.dupeBonusXp: 500`)
- 크리처 개체는 종당 1마리만 유지한다 (중복 개체를 따로 보관하지 않는다).

## 6. 샤이니

- 아무 크리처든 **최초 Lv.100 달성 시 샤이니 시스템이 계정 단위로 영구 해금**된다.
- 해금 후 모든 알은 부화 시 **5% 확률로 샤이니**로 태어난다. (`shiny.rate: 0.05`)
- 샤이니는 별도 아트가 아니라 **ANSI 색상 변이**로 표현하며, 도감에 ✦로 표기한다.
- 이미 보유한 종이 샤이니로 부화하면 해당 종을 샤이니로 승격하고(레벨·XP 유지) 중복 보너스는 지급하지 않는다.
- 스타터는 알에서 나오지 않으므로 1차 스코프에서 샤이니는 야생 3종만 가능하다.

## 7. statusline

한 줄 렌더. 형식:

```
(글리프) 이름 Lv.n [████░░░░] n% 🥚n%
```

- 글리프: 종별 1~2자 심볼 (아트 데이터에 정의).
- XP바: 8칸, 현재 레벨 진행률. 퍼센트는 정수 내림.
- `🥚n%`: 미부화 알 보유 시에만 표시 (부화 진행률).
- 진화·부화 발생 직후 **1회 한정** 짧은 축하 문구를 덧붙인다 (예: `✨ 진화!`).
- **절대 깨지지 않는다**: 파싱 실패·파일 손상·미지의 입력 시 예외를 던지지 말고 마지막 정상 상태를 렌더한다.
- 무거운 의존성·네트워크 호출 금지 (기동 속도).

## 8. CLI

| 명령 | 동작 |
|---|---|
| `init` | 스타터 선택(대화형), state 초기화, Claude Code settings.json에 statusline 등록 |
| `status` | 활성 크리처의 큰 ASCII 아트 + 레벨/XP/진행률/알 상태 상세 표시 |
| `dex` | 6종 도감. 미획득 종은 실루엣 + `???`, 샤이니는 ✦ 표기, 부화 횟수 표시 |
| `switch` | 보유 크리처 중 활성 크리처 교체(대화형 또는 ID 인자) |

## 9. 1차 스코프 제외

아래는 의도적으로 만들지 않는다. 요청이 있어도 이 문서를 먼저 개정하기 전에는 구현하지 않는다.

- 방치 패널티 (안 쓰면 퇴화 등)
- 서버 통신·싱크 (events.jsonl은 2차 싱크의 근간으로만 유지)
- 크리처 종 추가
- 재화/상점 — 나머지 스타터 2종 획득 수단 포함
- 크리처 닉네임(이름 짓기)
- 다중 알 보유

## 부록 A. 데이터 계약

### state.json

```json
{
  "version": 1,
  "activeId": "bitty",
  "creatures": {
    "bitty": { "level": 12, "totalXp": 1900, "shiny": false, "hatchCount": 0 }
  },
  "egg": { "xp": 23 },
  "shinyUnlocked": false,
  "tokenRemainder": 412,
  "encounterTokens": 3200,
  "sessions": { "sess-abc": 1234567 }
}
```

- `egg`는 미보유 시 `null`.
- `encounterTokens`: 다음 조우 굴림까지 누적된 토큰 (10,000 도달 시 굴림 후 차감).
- `sessions`: 세션 ID → 마지막으로 관측한 누적 토큰. XP는 이 값과의 델타로 계산한다. 최근 20개 세션만 유지.
- 저장은 temp 파일 write 후 rename으로 원자적으로 수행한다.

### events.jsonl

append-only. rewrite/삭제 금지. 모든 이벤트에 `version`, `ts`(ISO 8601), `type` 필수.

이벤트 타입: `xp_gain`, `level_up`, `evolve`, `egg_found`, `hatch`, `shiny_unlock`

### curve.json 기본값

```json
{
  "xp": { "tokensPerXp": 1000, "base": 10, "exponent": 1.5, "maxLevel": 100 },
  "evolution": { "gates": [10, 25, 40, 70] },
  "egg": {
    "rollPerTokens": 10000,
    "encounterRate": 0.02,
    "hatchXp": 50,
    "dupeBonusXp": 500,
    "speciesWeights": { "buggsy": 60, "daemo": 30, "kern": 10 }
  },
  "shiny": { "rate": 0.05 }
}
```

- 내장 기본값에 사용자 오버라이드 파일을 얕은 병합이 아닌 **키 단위 딥 병합**으로 적용한다.

## 부록 B. 테스트 필수 커버리지

- 경험치 곡선 (환산, 이월, 만렙 정지)
- 진화 관문 10/25/40/70 (다중 레벨업 관문 건너뛰기 포함)
- 알: 조우 확률 경계, 단일 알 제한, 부화 추첨, 중복 보너스
- 샤이니: 최초 만렙 해금, 부화 확률, 기존 종 샤이니 승격
